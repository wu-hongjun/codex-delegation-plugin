==================================================================
T15a FIX APPLIED — reconciler infers turn-completion from sidecar
==================================================================

Two interlocking bugs in packages/runtime/src/reconciler.ts:

1. mapStatus (line ~215-232): idle + queued → 'running' as a
   defensive fallback, never consulting sidecar evidence.

2. Turn-status mirror (line ~524-538): only flipped
   turn[last].status to 'completed' when nextStatus === 'completed',
   never on 'awaiting_followup' (even though awaiting_followup
   definitionally means the latest turn is completed).

Together: cmdDelegate writes turn[0].status='queued' and exits.
Reconciler later sees idle+queued, ignores sidecar.state=done,
returns 'running'. Mirror doesn't fire. Job stuck forever.

Fix:
  (a) mapStatus now checks sidecar.state==='done' &&
      sidecar.tempo==='idle' && sidecar.output.result is a
      non-empty string. If true, queued/working/starting/injecting
      are treated as effectively 'completed' for the mapping.
  (b) Turn-status mirror flips turn[last].status to 'completed'
      when nextStatus is either 'completed' OR 'awaiting_followup',
      and stamps endedAt if not already set.

Regression tests: 6 new tests in
  packages/runtime/test/reconciler.test.mjs describe block
  'T15a — sidecar-evidence-of-completion (plan 0002)'
  covering happy path (queued/working) + negative cases
  (no result, whitespace result, state=working, no-double-flip).

Local test impact: runtime 150→156 (+6 T15a).

T15a SECOND FIX — attach.ts warmup delay for real Claude TUI
==================================================================

First T15a fix unblocked the reconciler. Retrying $claude-followup
then failed with: 'follow-up prompt did not register within 5000ms'.

Direct PTY test confirmed: real Claude's TUI takes ~2-3s to finish
its startup banner + capability handshake before accepting input.
Without a warmup the PTY write lands during init and the TUI
ignores the keystrokes (no submit fires).

agents --json status stayed 'idle', timeline.jsonl gained no new
entry — Claude never received the prompt.

Fix: AttachAndSendOptions gains 'attachWarmupMs?' (default 2000).
  - Inserts an awaitable delay between pty.spawn('claude attach',
    ...) and term.write(prompt + '').
  - Env-var override CC_PLUGIN_CODEX_ATTACH_WARMUP_MS for tests
    (mock-claude responds instantly; tests set this to 0).
  - Defensive parse: bad input falls back to the 2000ms default.

with the env var set to 0 (no test-suite slowdown).

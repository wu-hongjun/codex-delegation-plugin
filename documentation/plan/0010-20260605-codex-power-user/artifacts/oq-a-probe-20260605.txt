## Verdict

**A** — `claude --bg "/goal <condition>"` successfully sets the goal as a slash command.

The load-bearing evidence is the JSONL record at line 8 of session 06a1cf7e: an
attachment with `"type": "goal_status", "sentinel": true, "condition": "example: respond
with the literal word OK and stop"` was written at session start, confirming that the
runtime parsed `/goal example: respond with the literal word OK and stop` as a slash
command and established the stop-condition before the first model call. Line 12 further
confirms a session-scoped Stop hook was injected into the context with the condition
text. By line 19, `"met": true` with `"iterations": 1` shows the goal completed in a
single turn. The TUI log independently showed `◎ /goal active` and `✔ Goal achieved`
banners. Probe 2 showed none of these markers — the positional prompt was simply treated
as user text, with no goal_status attachment and no Stop hook. The two probes are
structurally distinct: probe 1 entered goal mode; probe 2 did not.

**Implication for T3b**: `$claude-goal <condition>` can be implemented as a thin wrapper
that calls `claude --bg "/goal <condition>"` via `_runDelegateCore` with prompt
transformer `p => '/goal ' + p`, exactly mirroring the `$claude-workflow` pattern. No
PTY injection or `claude attach` flow is needed. This is Outcome A.

## Verdict

**B** — `/tasks` is a TUI-only interactive command; it cannot be driven via `claude --bg`.

The load-bearing evidence is the `claude agents --json` record showing `"status": "waiting",
"waitingFor": "dialog open"` for session 3a78ef70 — the command opened an interactive
picker dialog and blocked indefinitely waiting for keyboard input. No JSONL conversation
file was created (the session never progressed to a model turn), and the `claude logs`
output confirmed a fully rendered TUI task-picker panel awaiting user interaction.
By contrast, probe-2 (sanity) completed normally with the model responding "OK" in
a standard conversation record. The two sessions are structurally distinct: probe 1 is
a blocked TUI dialog; probe 2 is a completed background conversation.

**Implication for Plan 0011 T2-T5**: `/tasks` is deferred. No SKILL.md, no dispatcher
`case 'tasks':`, no count bumps for this command. Defer to Plan 0012+ with PTY-injection
fallback design.

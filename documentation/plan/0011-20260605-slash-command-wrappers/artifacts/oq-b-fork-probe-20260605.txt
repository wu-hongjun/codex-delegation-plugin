## Verdict

**A** — `/fork` is parsed as a slash command via `claude --bg`; it spawns a real subagent.

The load-bearing evidence is JSONL L12: a completed `<task-notification>` record with
`<status>completed</status>`, `<result>OK</result>`, and a 20,990 ms subagent run that
consumed 30,117 tokens — proving the runtime parsed `/fork <directive>` as a slash
command and dispatched the directive to a new forked subagent process. The subagents/
directory containing `agent-arespond-with-the-ccdae9dbf49fafdd.jsonl` independently
confirms a full subagent conversation was written. The session name being the literal
slash command string (not an AI title) is consistent with /goal's behavior. By contrast,
probe-2 (sanity) produced a normal single-turn conversation with an AI-generated session
name and no task-notification record.

**Implication for Plan 0011 T2-T5**: implement `$claude-fork`. The prompt transformer
is `p => '/fork ' + p`, mirroring `$claude-goal`. The directive is the skill argument.
Note: subagent spawning is async — the parent session completes when the subagent
finishes; `_runDelegateCore` should handle this the same way as `$claude-goal`.
Token cost is higher than /goal (30k tokens for a trivial directive) — document in
SKILL.md.

CONTEXT
-------
OQ-C asks: when "/deep-research X" is sent via $claude-delegate (cmdDelegate, which uses
promptTransformer = p => p — identity, no prefix), does the slash command still get parsed
as a slash command (XML form), or does it arrive as plain text?

This is the key question for the saved-workflow invocation shape: if cmdDelegate preserves
the "/" prefix intact and claude --bg parses it as a slash command, then users can invoke
any saved /<name> workflow via $claude-delegate -- "/<name> <args>".

SIDE EVIDENCE
-------------

1. The JSONL for session 2c00730b is structurally identical to session b60f2f86 (OQ-B direct
   invocation): same slash-command parse XML, same workflow definition injection, same
   assistant confirmation. The only difference is the session name
   ("codex:cc-plugin-codex:mq2wu3fi" vs "/deep-research What is 2 plus 2?") due to cmdDelegate
   setting a Codex-namespaced session name.

2. cmdDelegate's promptTransformer = (p) => p (identity). The "/" prefix is passed through
   unchanged. This contrasts with:
   - cmdWorkflow: promptTransformer = (p) => `ultracode: ${p}`  (adds ultracode: prefix)
   - cmdGoal:     promptTransformer = (p) => `/goal ${p}`       (adds /goal prefix)
   - cmdFork:     promptTransformer = (p) => `/fork ${p}`       (adds /fork prefix)
   - cmdBatch:    promptTransformer = (p) => `/batch ${p}`      (adds /batch prefix)

3. claude --bg parses any prompt starting with "/" as a slash command at the protocol level,
   regardless of whether it arrived via direct invocation or through a driver/dispatcher.
   This is a property of the claude CLI itself, not of the dispatcher.

4. For saved workflows (created via /workflows → press 's' in TUI), the same mechanism
   applies: $claude-delegate -- "/<saved-name> <args>" will cause claude --bg to parse
   "/<saved-name>" as a slash command and invoke the saved workflow runtime.


VERDICT
-------
A — cmdDelegate sends the "/" literally intact; claude --bg parses it as a slash command,
    identical to direct invocation. Saved-workflow invocation via $claude-delegate confirmed.

Justification:
  - The JSONL user turn for the cmdDelegate probe shows the parsed XML slash-command form
    (<command-name>/deep-research</command-name>), not plain text, confirming slash-command
    parse is active.
  - cmdDelegate's identity promptTransformer passes the raw prompt unchanged; claude --bg
    parses any "/" prefix as a slash command at the protocol level.
  - The workflow runtime engagement (definition injection, extended thinking, model
    confirmation) is identical between direct invocation (OQ-B) and cmdDelegate routing (OQ-C).

Saved-workflow invocation path conclusion for T4:
  Confirmed invocation shape: $claude-delegate -- "/<saved-name> <args>"
  - cmdDelegate's identity transformer passes the slash-command string verbatim.
  - claude --bg parses it as a slash command.
  - The workflow runtime receives it via the standard <command-name>/<command-args> XML path.
  - $claude-workflow must NOT be used for saved-workflow invocations (it would prepend
    "ultracode: " to the prompt, breaking the slash-command parse).
  - T4 documentation should specify $claude-delegate -- "/<name> <args>" as the canonical
    invocation shape for saved/bundled workflows.

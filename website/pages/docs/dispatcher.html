Dispatcher CLI reference

Codex skills are the normal interface. The dispatcher is useful for scripts, diagnostics, contributor testing, and exact action hints returned by JSON output.

Choose the correct path

Installed public plugin

codex plugin list
node ~/.codex/plugins/cache/codex-delegation-plugin/delegate/{{version}}/scripts/delegate.mjs --version

Repository checkout

node packages/plugin-delegate/scripts/delegate.mjs --version

Use the installed version reported by codex plugin list or the exact meta.dispatcherPath returned by JSON commands. The plugin upgrade flow also creates a stable current symlink, but a fresh codex plugin add does not guarantee that link exists.

The dispatcher has no help subcommand. Use --help globally or after a command.

Examples

node packages/plugin-delegate/scripts/delegate.mjs setup
node packages/plugin-delegate/scripts/delegate.mjs doctor --claude-access --json
node packages/plugin-delegate/scripts/delegate.mjs delegate -- "Inspect this repository."
node packages/plugin-delegate/scripts/delegate.mjs status --job <jobId> --json --compact
node packages/plugin-delegate/scripts/delegate.mjs wait <jobId> --timeout 5m --json --compact
node packages/plugin-delegate/scripts/delegate.mjs result <jobId>
node packages/plugin-delegate/scripts/delegate.mjs followup <jobId> -- "Continue with the tests."
node packages/plugin-delegate/scripts/delegate.mjs stop <jobId>
node packages/plugin-delegate/scripts/delegate.mjs delegate --provider agy --mode plan -- "Inspect this repository."
node packages/plugin-delegate/scripts/delegate.mjs status --provider agy

JSON and action hints

Use --json --compact for automation around delegate, status, wait, result, and stop. Compact payloads include normalized state, provider, result state, blocked reason, and exact recovery commands. Follow the returned action hints instead of reconstructing internal paths.

A JSON wait timeout can include timeoutRecovery with commands for status, partial result, stop, or Claude attach. A timeout is an expired waiting budget, not proof that the job died.

Prompt boundary

Put dispatcher flags before -- and prompt text after it. This prevents prompt words beginning with dashes from being parsed as flags. 

delegate delegate --model claude-opus-4-8 -- "Review --fix references as plain prompt text."

Exit behavior

  • Invalid command or flag syntax exits non-zero and prints usage or a targeted error.
  • A wait timeout exits non-zero but can still return the latest JSON status payload.
  • Review gates can exit 1 after printing findings when --blocking or --fail-on trips.
  • Provider launch and supervision failures remain represented in the job store for inspection.

Safety, privacy, and configuration

Delegation crosses a provider boundary. The plugin makes that boundary explicit.

Workspace and provider disclosure

A delegated task may send repository contents, prompts, command output, and file metadata to Anthropic or Google through the selected local provider account. The first delegation to each provider in each workspace requires acknowledgement.

Acknowledgements live in ~/.codex/codex-delegation-plugin/acks/. Approval for Claude Code does not approve Antigravity, and approval in one repository does not approve another. Use --yes only for intentional non-interactive acknowledgement.

Claude Code permissions

Normal jobs use Claude Code’s permission system. For a specifically trusted unattended job, the user can request --bypass-permissions, --permission-mode bypassPermissions, or --dangerously-skip-permissions. The dispatcher forwards Claude Code’s literal bypass flag and pairs it with the required acknowledgement when explicitly requested.

Bypass does not select a Chrome profile, complete a passkey, inspect secret browser stores, or replace a local user gesture. Do not infer unattended permission from an ordinary delegation request.

Antigravity permissions and sandbox

Antigravity print mode cannot show an interactive command-permission prompt. Jobs that need terminal commands require either a narrow Antigravity permissions.allow rule or an explicit trusted --dangerously-skip-permissions run.

--sandbox enables Antigravity’s terminal sandbox. It restricts execution but does not grant command permission. See the official Antigravity permission reference.

Exact follow-up is turn-scoped, not interactive. The plugin cannot answer a permission prompt while an Antigravity print turn is running. A denied operation fails the turn and is recorded as a failed job instead of a successful empty response.

Antigravity native control-plane limits

The headless CLI supports targeted conversation resume but does not expose live TUI attach, mid-turn input injection, the native /fork picker, or nested-subagent detail panels. The plugin does not scrape or automate those interfaces. Use the interactive Antigravity TUI when those controls matter.

Real Chrome handoff

Use --chrome on a fresh Claude job that needs the connected Chrome extension flow. This is separate from Codex’s in-app browser. If Claude needs profile selection, login, passkey, or another user gesture, inspect compact status and follow its waiting.userAction instruction, usually claude attach <shortId>.

--allow-edit is not a sandbox

The flag communicates task policy or framing to the wrapper. It does not bypass the privacy acknowledgement, change provider permissions, or enforce filesystem isolation. Review and orchestration commands that do not use this framing reject it.

State root

Runtime state defaults to ~/.codex/codex-delegation-plugin/. The runtime also honors CODEX_DELEGATION_HOME as an alternate root; the repository test suite uses it to isolate fixtures from real jobs. Avoid pointing it at a shared or disposable directory unless that isolation is intentional.

Other CODEX_DELEGATION_* variables in the source support tests, timeouts, or packaging internals and are not a stable end-user configuration API unless documented by the command that uses them.

Workspace scope

Status and job actions default to the current workspace. Use --all only when you intentionally want cross-workspace resolution. For a cross-workspace follow-up, privacy is checked against the target job’s workspace, not the caller’s current directory.

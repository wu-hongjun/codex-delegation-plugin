Common flags

--json
Return machine-readable output where the selected command supports it.
--compact
Use the smaller redacted JSON shape for job-oriented commands.
--all
Resolve jobs across all workspaces instead of only the current workspace.
--yes
Record the provider privacy acknowledgement non-interactively when supported.
--name <name>
Give a fresh provider session a readable label. It is not a resume key.
--model, --effort, --agent
Select provider launch behavior for fresh sessions.
--add-dir <path>
Add another allowed workspace root to a fresh job. Repeat when needed.
--chrome
Ask a fresh Claude session to use real Chrome. There is no delegate --real launch flag.
--bypass-permissions
Explicit trusted unattended mode. See Safety.
--allow-edit
A policy/framing signal only. It is not a filesystem sandbox or permission bypass.

Individual skills intentionally accept different subsets. Use the syntax shown above or the dispatcher help; invalid startup-only flags are rejected with a targeted error.

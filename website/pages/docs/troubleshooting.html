Troubleshooting

Start from the plugin’s probes and action hints; avoid editing state records by hand.

Setup reports warn or fail

$claude-setup
$claude-doctor --claude-access --json
$agy-setup --json
  • warn: read the named probes. Optional daemon or deferred auth checks can warn without blocking delegation.
  • fail: fix the failed binary, auth, model access, workspace, or capability probe before launching a job.
  • Run doctor before long, real-browser, or unattended Claude work.

A job needs input

$claude-status --job <jobId> --json --compact

Read operatorState, blockedOn, and the exact action hints. For a normal permission or browser selection, use the returned claude attach <shortId> instruction. For a trusted unattended retry, explicitly choose the returned restart-with-bypass action and provide a fresh complete prompt. 

$claude-stop --all-needs-input
$claude-stop --all-needs-input --all

Wait timed out

A timeout does not stop the job and does not mean it is unhealthy.

$claude-status --job <jobId> --json --compact
$claude-result <jobId> --partial
$claude-wait <jobId> --timeout 10m --json --compact

JSON wait output includes timeoutRecovery. Use those exact status, partial-result, stop, or attach commands.

Result says the job is incomplete

This protects automation from treating partial text as final. Wait for a result state or ask explicitly for the latest artifact: 

$claude-result <jobId> --partial
$agy-result <jobId> --partial

Provider logs disappeared after exit

Use plugin-owned status, wait, result, and partial-result artifacts after a provider process exits. Raw provider logs are diagnostic and may be transient or contain terminal control sequences.

Skills or version look stale

$claude-upgrade
$claude-upgrade --yes

Manual public-install equivalent:

codex plugin marketplace upgrade "codex-delegation-plugin"
codex plugin remove "delegate@codex-delegation-plugin"
codex plugin add "delegate@codex-delegation-plugin"
codex plugin list

Restart Codex after reinstall. For immediate diagnostics, use the version reported by codex plugin list or the exact meta.dispatcherPath from JSON output: 

node ~/.codex/plugins/cache/codex-delegation-plugin/delegate/{{version}}/scripts/delegate.mjs --version

$claude-upgrade --yes creates a stable current symlink as part of its refresh flow; do not assume the symlink exists after a fresh marketplace install.

Uninstall

codex plugin remove "delegate@codex-delegation-plugin"
codex plugin marketplace remove "codex-delegation-plugin"

This removes plugin registration and the marketplace pointer. It intentionally does not delete ~/.codex/codex-delegation-plugin/ job records or provider-owned transcripts.

Report a defect

Include plugin version, provider version, the failing command, sanitized setup or status JSON, and a minimal reproduction. Do not attach secrets, private prompts, raw auth data, or repository contents that should not be public.

Workflows and reviews

Use Claude Code’s live sessions for iterative work, orchestration, and structured evaluation.

Continue a completed turn

$claude-delegate "Inspect the authorization layer. Report only."
$claude-wait <jobId> --json --compact --timeout 5m
$claude-followup <jobId> -- "Now trace the highest-risk path to its tests."
$claude-result <jobId>

Follow-up attaches to the original live Claude session. It cannot change startup-only model, workspace, or MCP configuration. If permission handoff cannot happen non-interactively, the command fails closed and returns a manual claude attach <shortId> instruction.

Dynamic workflow

$claude-workflow -- "Audit every fetch call and migrate the unsafe call sites in verified stages."
$claude-workflows
$claude-workflows <jobId> --json

Workflow injects the ultracode: planning trigger. The read-only workflows command lists plugin-owned workflow and deep-research records; it is separate from the Claude TUI’s session-local /workflows panel.

Goal, fork, and batch

Goal condition

$claude-goal -- "All unit tests pass and no source file contains the deprecated API name."

Focused fork

$claude-fork -- "Build a proof of concept for the new rate limiter and report tradeoffs."

Parallel batch

$claude-batch -- "Migrate all independent packages from the old API to the new API, then run their tests."

These commands create fresh sessions. Batch can cause several parallel tool calls or subagents, so its cost grows with the number of tasks and affected files.

Deep research

$claude-deep-research -- "What are the current tradeoffs between B-trees and LSM-trees for write-heavy workloads?"

Deep research starts a Claude workflow with multi-agent web search and citation cross-checking. Use it for research questions, not ordinary repository inspection.

Choose a review mode

Review mode comparison
Mode Session Best for
$claude-review Reuses the live original session Fast, context-rich checks while the session is available
$claude-adversarial-review Starts a fresh independent session Thorough second opinions and terminal jobs with a stored result 
$claude-review <jobId> --json
$claude-adversarial-review <jobId> --blocking

Both return a verdict and severity-rated findings. Use --blocking as the shorthand high/blocker/fail gate, or --fail-on fail|any|nit|low|medium|high|blocker for an exact automation threshold.

Restart a blocked job

Compact status can return actionHints.restartWithBypass for an explicitly trusted unattended retry. Restart needs a fresh prompt because job records keep prompt metadata, not the full original prompt text. 

node packages/plugin-delegate/scripts/delegate.mjs restart <jobId> --bypass-permissions --yes -- "Fresh complete prompt"

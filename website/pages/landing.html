01 / Operating model

The agent does the work. Codex keeps the controls.

Provider-native execution stays intact. The plugin adds just enough structure to start, inspect, continue, review, and stop work deliberately.

Control

Codex stays responsive

Lengthy investigation and implementation continue in a supervised background job.

Record

Every job is inspectable

A stable ID ties together state, logs, results, action hints, and provider metadata.

Adapter

Provider behavior stays honest

Claude sessions and Antigravity print jobs expose only the actions they truly support.

02 / Lifecycle

Start once. Return when judgment is needed.

Start with a bounded instruction. Later Codex turns can wait, inspect partial or final output, continue a live Claude session, or stop work that no longer matters.

04 / Boundary

A control plane, not a terminal scraper.

Runtime state comes from provider-native sessions and plugin-owned records. Session labels never masquerade as durable job identifiers. 

Codex
  └─ delegate: skill

          ├─ ClaudeBackgroundDriver ── live Claude session
          └─ AgyCliDriver ─────────── supervised agy process
                 │
                 └─ job record · events · logs · results · next actions

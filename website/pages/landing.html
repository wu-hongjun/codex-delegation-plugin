Delegate the work. Keep Codex in control.

Codex Delegation is a Codex-native plugin that starts, supervises, and inspects coding jobs running through your local Claude Code or Google Antigravity account.

Install and run a first job · Read the documentation · View release v{{version}}

Install

codex plugin marketplace add https://github.com/wu-hongjun/codex-delegation-plugin
codex plugin add "delegate@codex-delegation-plugin"
codex plugin list

Restart Codex after installation so the delegate: skill namespace is loaded.

What it does

  • Starts Claude Code as a first-party background session or Antigravity as a supervised agy --print process.
  • Records a provider-neutral job ID, state, logs, and results under ~/.codex/codex-delegation-plugin/.
  • Lets later Codex turns check status, wait, read partial or final output, stop work, and—when supported—continue or review the same session.

The operator loop

$claude-delegate "Inspect this repository and identify the highest-risk area. Do not edit files."
$claude-wait <jobId> --json --compact --timeout 5m
$claude-result <jobId>
$claude-followup <jobId> -- "Now propose the smallest safe fix."
$claude-stop <jobId>

Swap the prefix for $agy-* when a single-turn Antigravity job is a better fit.

Two providers, one job model

Capability Claude Code Google Antigravity
Transport Native background session Supervised agy --print process
Follow-up Yes, while the session remains live No; start a new single-turn job
Status, wait, result, stop Yes Yes
Advanced workflows and reviews Yes Not exposed by print mode

Codex-first architecture

Codex
  └─ delegate: skill
      └─ delegate dispatcher
          ├─ ClaudeBackgroundDriver ── Claude Code background session
          └─ AgyCliDriver ─────────── supervised agy process
                 │
                 └─ job store: status, logs, results, action hints

The runtime uses provider-native state and plugin-owned artifacts. It does not scrape a human terminal interface, and it does not treat a session name as a reusable job ID.

Why use it

  • Keep Codex available while longer work continues in the background.
  • Give each delegated task a stable, inspectable lifecycle.
  • Use explicit privacy acknowledgement and permission boundaries per workspace and provider.
  • Choose a provider without changing the status, wait, result, and stop workflow.
  • Run same-session or independent review without losing the original job record.

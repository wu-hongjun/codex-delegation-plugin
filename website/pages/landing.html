One job model

Codex Delegation gives provider-native work a stable, inspectable lifecycle. Each job keeps its ID, state, logs, result, and available next actions under ~/.codex/codex-delegation-plugin/.

  • Claude Code runs as a first-party background session.
  • Antigravity runs as a supervised agy --print process.
  • Both expose the same status, wait, result, and stop vocabulary to Codex.

The operator loop

  1. Delegate a bounded task$claude-delegate "Audit the highest-risk area. Do not edit files."
  2. Wait without blocking Codex$claude-wait <jobId> --json --compact --timeout 5m
  3. Inspect the recorded result$claude-result <jobId>
  4. Continue the same Claude session$claude-followup <jobId> -- "Propose the smallest safe fix."
  5. Stop when the work is no longer useful$claude-stop <jobId>

Use the equivalent $agy-* skills for a single-turn Antigravity job.

Codex-first architecture

Codex
  └─ delegate: skill
      └─ delegate dispatcher
          ├─ ClaudeBackgroundDriver ── Claude Code background session
          └─ AgyCliDriver ─────────── supervised agy process
                 │
                 └─ job store: status, logs, results, action hints

The runtime uses provider-native state and plugin-owned artifacts. It does not scrape a human terminal or confuse a session name with a reusable job ID.

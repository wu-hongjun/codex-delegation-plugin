01 / Operating model

The agent does the work. Codex keeps the controls.

Provider-native execution stays intact. The plugin adds just enough structure to start, inspect, continue, review, and stop work deliberately.

Control

Codex stays responsive

Lengthy investigation and implementation continue in a supervised background job.

Record

Every job is inspectable

A stable ID ties together state, logs, results, action hints, and provider metadata.

Adapter

Provider behavior stays honest

Each transport exposes only the lifecycle and advanced actions it can actually perform.

02 / Lifecycle

Start once. Return when judgment is needed.

Start with a bounded instruction. Later Codex turns can wait, inspect partial or final output, continue an exact provider conversation, or stop work that no longer matters.

Exact means job-scoped.

Follow-up uses the session ID captured for that job—never a provider-wide --continue. Pi and Qwen run headlessly, so they intentionally do not advertise live attachment or interactive permission handoff.

04 / Boundary

A control plane with a native escape hatch.

Structured state comes from provider sessions, hooks, transcripts, and plugin-owned records. The raw Antigravity TUI remains available when a human needs its native controls.
